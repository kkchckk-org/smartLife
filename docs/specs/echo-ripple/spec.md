# エコー波紋

## 目的

- ユーザーが水面をなぞった後、少し遅れて「誰か別の存在」がなぞり返したかのような波紋を生む
- 直接的な因果（自分の指 → 波紋）ではなく、間接的な応答（自分の指 → 沈黙 → 向こう側からの波紋）を体験させる
- 「...ん？今、何かが触った？」という微かな違和感＝生命感を生む

## スコープ

### 今回やること
- ユーザーのタッチ軌跡を一定時間バッファし、遅延再生する「エコー」を実装する
- エコーは空間的・時間的にずらし、ユーザー自身の軌跡の正確な再現にはしない
- 既存の Creature インターフェースに準拠した新クラスとして実装する

### 今回やらないこと
- エコーの視覚的な見た目の差別化（色や透明度の違い等）。波紋の物理は既存の `water.disturb` をそのまま使う
- 複数回のエコー（反復応答）。1回のなぞりに対して1回のエコーのみ
- エコー同士の相互作用や学習的な振る舞い

## 振る舞い

### 1. 軌跡の記録

ユーザーが `active: true` の間、タッチ座標を一定間隔（50ms ごと）でリングバッファに記録する。

- 記録する情報: `{ x, y, force }` （force = pressure * 30 + speed * 0.08、main.ts の既存計算と同じ）
- バッファサイズ上限: 直近 3 秒ぶん（約 60 サンプル）
- `active: false` になった瞬間（指が離れた瞬間）に記録を確定し、エコー再生の予約を行う

### 2. エコーの発動タイミング

指が離れてから **0.8 〜 1.6 秒** のランダムな遅延の後にエコー再生を開始する。

- 遅延時間は毎回ランダムに決定する（均一分布）
- 「考えてから返す」間が生命感の核心。短すぎると機械的、長すぎると因果が切れる

### 3. エコーの空間変換

記録した軌跡をそのまま再生せず、以下の変換を加える。

| パラメータ | 値 | 意図 |
|---|---|---|
| 平行移動 | 軌跡の重心から 30〜80px ランダムにずらす（方向もランダム） | 「同じ場所」ではなく「近くの別の場所」から応答が来る感覚 |
| スケール | 0.6 〜 0.9 倍 | エコーは元の動きより小さい。控えめに返す |
| 時間伸縮 | 1.1 〜 1.4 倍（ゆっくり再生） | わずかにたゆたうような、急がない動き |
| 力の減衰 | 元の force の 0.3 〜 0.5 倍 | 残滓であり、はっきりとは見えない |

各パラメータはエコー発動時に1回だけランダム決定する（再生中は固定）。

### 4. エコーの再生

- 記録したサンプル列を、変換済みの座標と force で順番に `water.disturb()` を呼ぶ
- 再生間隔は元の記録間隔（50ms）に時間伸縮係数を掛けたもの（55〜70ms 程度）
- 再生中にユーザーが新たにタッチした場合、**再生中のエコーは即座にフェードアウトする**（残り force を 0.3 秒かけてゼロに減衰）。ユーザーの入力が優先
- 1回の再生が終わったら、そのエコーは消滅する

### 5. 同時存在数の制限

- エコーは最大 1 つだけ存在する（キューイングしない）
- 新しいエコー予約が入った場合、古い予約は破棄する
- 複数の存在が一斉に動くと「仕組み」に見えてしまう。1体だけが、ひっそりと

## アーキテクチャ

### 新規ファイル: `src/creatures/echo.ts`

Creature インターフェースを実装する。理由:

1. 依存方向を守れる（`input → creatures → physics` の creatures レイヤーに収まる）
2. 既存の creature と同じライフサイクル（`update(dt, input, water)`）で動作する
3. main.ts の creatures 配列に追加するだけで統合できる

### 内部状態

```
EchoCreature {
  // 軌跡バッファ
  trailBuffer: Array<{ x, y, force }>
  recordTimer: number          // 記録間隔カウンタ
  recording: boolean           // 記録中フラグ
  wasActive: boolean           // 前フレームの input.active（立ち下がり検出用）

  // エコー再生
  echoScheduled: boolean       // エコー予約済みフラグ
  echoDelay: number            // 残り遅延時間
  echoSamples: Array<{ x, y, force }>  // 変換済みサンプル列
  echoIndex: number            // 再生位置
  echoInterval: number         // 再生間隔（変換後）
  echoTimer: number            // 次サンプルまでのカウンタ
  fadeOut: number              // フェードアウト係数 (1.0 → 0.0)

  // Creature インターフェース用（エコー再生中の現在位置）
  x, y: number
}
```

### update(dt, input, water) の処理フロー

```
1. input.active が true → 軌跡を記録（50ms間隔でサンプリング）
2. input.active が false に変わった瞬間 → エコーを予約（遅延・変換パラメータを決定）
3. エコー予約中 → 遅延カウントダウン
4. 遅延が 0 になったら → エコー再生開始
5. エコー再生中 → 間隔ごとに water.disturb() を呼ぶ
   - ユーザーが新たにタッチ → fadeOut 開始
6. 全サンプル再生完了 or fadeOut が 0 → エコー終了
```

### main.ts の変更

- `createEcho()` を import し、creatures 配列に追加する
- EchoCreature は1つだけ生成する（既存の3体の creature に加えて4体目）
- 既存のタッチ → `water.disturb` のロジックは変更しない

## 完了条件

1. **水面をなぞって指を離すと、約1秒後に近くの場所で控えめな波紋が自然に現れる**
2. **エコー波紋は元のなぞりより小さく・ゆっくりで、毎回微妙に異なる位置と大きさで再生される**
3. **エコー再生中に再度タッチすると、エコーが静かに消えてユーザーの操作が優先される**
4. **既存の生き物の水面押し上げ・タッチ追従に影響がない**
